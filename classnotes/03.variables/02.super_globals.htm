Super Globals

Superglobals are built-in variables that are always available in all scopes

$GLOBALS
An associative array containing references to all variables which are currently defined in the global scope of the script
$_SERVER
An array containing information such as headers, paths, and script locations
$_GET
An associative array of variables passed to the current script via the URL parameters
$_POST
An associative array of variables passed to the current script via the HTTP POST method
$_FILES
An associative array of items uploaded to the current script via the HTTP POST method
$_COOKIE
An associative array of variables passed to the current script via HTTP Cookies
$_SESSION
An associative array containing session variables available to the current script
$_REQUEST
An associative array that by default contains the contents of $_GET, $_POST and $_COOKIE
$_ENV
An associative array of variables passed to the current script via the environment method

note: most Super globals were once referred to as HTTP_*_VARS so, HTTP_SESSION_VARS, HTTP_GET_VARS but this is now depreciated and the more modern equivalents should be used.

// echos a name if it exists
if ( isset() $_GET['name'] ) ) echo 'Hello ' . $_GET['name'];

$_GET

The $_GET super global is built from variables passed using the URL. They are usually sent as key name pairs following the page (if present) and a question mark, separated by an ampersand

// URL: http://website.com/index.php?name=bob&amp;age=32
echo $_GET['name']; // bob
echo $_GET['age']; // 32

$_POST

$_POST works in a similar way to $_GET but the data is sent as a separate packet and not through the URL. This is how form values are sent using the post form method. It uses the name attribute to name the variable.

<form action="" method="post">
<input type="text" name="name">
</form>

// once submited the value is available
echo $_POST['name']; // value set in the form

$_SERVER

The entries in this array are created by the web server. There is no guarantee that every web server will provide any of these; servers may omit some, or provide others not listed here. Most of the time most values are set.

Here are some useful entries but there are many more.

PHP_SELF
The filename of the currently executing script
REQUEST_METHOD
Which request method was used to access the page; i.e. 'GET', 'HEAD', 'POST', 'PUT'.
HTTP_REFERER
The address of the page (if any) which referred the user agent to the current page. This is set by the user agent
HTTP_USER_AGENT
Contents of the User-Agent: header from the current request. Usually the browser name.
REMOTE_ADDR
The IP address from which the user is viewing the current page
REQUEST_URI
The URI which was given in order to access this page; for instance, '/index.php'
