Comments

When to comment

When code is very dense
Sometimes having multiple nesting of functions or if statements can be hard to follow
When there are lots of abbreviations
Shorten and temporary variables might cause confusion
When using an uncommon function or technique
Not everyone knows everything - outline anything others might not have seen before
When something is unclear
Think of it as a very brief description

Over-commenting

Sometimes comments just aren't needed. If the code is obvious then there is no need to clutter the code.

Keep comments short

Be brief and clear.

Provide links

Link to other libraries of code if you use them. This can be useful for documentation and updates.

But remember good code is better than good comments.
