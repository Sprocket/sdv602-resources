Dates

date

Returns a string formatted according to the given format string using the given integer timestamp or the current time if no timestamp is given.

The date format uses characters that have a special meaning. By rearranging them you can specify the date format you require:

date("Y-m-d"); // 2012-08-16
date("d/m/Y"); // 16/08/2012

You can also get time:

date("h:i:s A"); // 08:25:52 PM
date("G:i:s");  // 20:25:52

Months and days can be formatted in several ways:

date("F"); // August
date("M"); // Aug
date("l");  // Thursday
date("D");  // Thu

You can also get Ante meridiem and Post meridiem and the ordinal suffix

date("F j, Y, g:i a"); // August 16, 2012, 8:31 pm
date("dS");  // 16th

There are also some common formats:

date("c"); // 2012-08-16T20:28:24+12:00 (ISO 8601 date)
date("r"); // Thu, 16 Aug 2012 20:28:24 +1200 (RFC 2822 formatted date)

If you want to use characters within the template you have to escape them:

date('\i\t \i\s \t\h\e jS \d\a\y'); // it is the 16th day

You can use date functions within tests and incorporate them into strings:

if ( date("L") ) echo 'It is a leap year'; // It is a leap year
echo 'It is week number ' . date("W"); // It is week number 33

strtotime

Parse about any English textual datetime description into a Unix timestamp

strftime

strftime("%B %d, %Y", time()); // July 28, 2012
