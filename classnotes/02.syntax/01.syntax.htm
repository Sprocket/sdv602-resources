PHP Syntax

(http://www.php.net/manual/en/language.basic-syntax.php 'PHP Manual on Basic Syntac')

PHP is derived from C and its syntax is similar to C, C++, C#, Perl, Java and Javascript.

Opening and Closing Tags

The delimeters or 'on' and 'off' tags in PHP are: <?php and ?> You then put your code between them, like this

<?php echo 'hello world'; ?>

There are variations on these but they are not reccomended as support is not consistent.

Each statement ends in a semicolon. It is conventional to have each statement on its own line.

Embeding PHP in HTML

PHP was created as a sort of templating language for HTML. You can disperse them amongst HTML.

A page with a .php extension will be parsed by the PHP engine before being sent to the client. Everything within PHP tags will be parsed - the code will be executed and the results simply inserted into the HTML page.

Everything outside of the PHP tags is ignored by the parser and simply sent to the browser, allowing you to have mixed content.

You can use as many PHP tags on a single page as required.

Whitespace

This is not considered significant and is simply collapsed into a single space, this gives you the freedom to layout your code as you see fit.

Comments

The PHP engine ignores comments - they're meant for humans to read. Use them to

  • explain what you are doing when it is not obvious or unconventional
  • make a note to others; mark a TODO, FIXME, etc.
  • temporary disable code when debugging

There are three types of comments in PHP. Two single line comments that work on single lines:

// This comment will run across one line (generally preferred)
# As will this one

And mutli-line comments

/*
This comment will run across
several lines
*/
